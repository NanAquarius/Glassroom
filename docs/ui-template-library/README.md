# Glassroom UI template library

This directory stores **UI reference patterns** for Glassroom case pages.

The purpose is not to clone any one source page, but to preserve reusable interface patterns that work well for this course domain.

## How to use this library

When designing or revising a Glassroom case page, choose the template that best matches the case structure:

- `teaching-case-analysis.json`
  - balanced teaching page
  - good default for most course cases

- `evidence-led-investigation.json`
  - stronger evidence-map emphasis
  - good when source conflict or provenance matters most

- `timeline-first-case.json`
  - stronger chronology emphasis
  - good when pattern failure, recency bias, or sequence confusion is central

- `guided-cognitive-trap-lab.json`
  - guided split-view teaching page
  - good when the source document and the analysis rail should remain visible together
  - derived from a de-identified classroom artifact and preserved as a fixed UI reference pattern

## Rule

These templates are **reference patterns**, not rigid page skins.

Glassroom should learn from them, then adapt:

- section ordering
- card density
- evidence framing
- analysis flow
- teaching readability

instead of copying surface details.

For de-identified reference pages stored under `reference-pages/`, preserve reusable instructional structure while removing personal, instructor, and course-owner identity markers.
